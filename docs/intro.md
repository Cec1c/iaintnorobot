# 项目介绍

I Ain't No Robot 是一个 AstrBot 插件原型，用来让机器人在已启用群聊里低频、短句、自然地插话。

设计重点是克制：

- 不把每条消息都交给 LLM。
- 不把群聊完整历史当知识库存起来。
- 不输出长段解释、列表、总结或客服式回答。
- 不承认自己是 AI、机器人、模型或插件。
- 被说像 AI、被骂 AI、被问插件行为时，可以不理，也可以像普通群友一样短怼或糊弄过去。
- WebUI 中只在已添加且启用的群聊里运行，避免插件跑到未授权群。

## 工作流

```text
群消息进入
  -> 插件记录短期纯文本
  -> 后台定时醒来
  -> 本地规则判断是否活跃、是否太频繁
  -> 优先判断有没有可附和或可讨论的话题
  -> 随机延迟 3 到 120 秒
  -> 重新读取最新聊天
  -> 必要时调用 AstrBot 当前 LLM
  -> 清洗输出
  -> 合格才发一句
```

群内直接艾特机器人时，插件作为低优先级兜底处理：其他插件和命令先运行；如果消息命中放行规则，例如 `/在线` 或 `重启服务器`，本插件不会接管。普通聊天式艾特会立即给一句短回，并默认阻止 AstrBot 的标准 LLM 会话继续处理。主动插话才走随机延迟。

## 记忆策略

插件只做轻量记忆：

- 最近消息：默认每群最多 200 条，且只存纯文本。
- 群话题摘要：例如最近常聊什么。
- 群语气摘要：例如随意、嘴碎、爱吐槽、游戏群等。
- 黑话释义：只把已理解的黑话喂给发言模型。
- 内线问题：不确定黑话会按冷却交给 LLM 生成私信去问内线，没回就保持不确定。
- 机器人发言记录：避免短时间重复说同一句。
- 表达线索：压缩自己最近的态度/观点，下一次短句可以顺着说。

SQLite 文件位于 AstrBot 插件数据目录中。它不是外部服务，不需要用户额外部署数据库。

## 输出风格

输出目标是“像真人懒得打字”：

- 一句话。
- 字数短。
- 不自证。
- 不装懂。
- 能闭嘴就闭嘴。
- 可以半截话、轻吐槽、短附和。
- 有话题先接话题，没话题再低概率自创一句。
- 自创句子偏模糊感受或情绪碎念，例如“ok今天已到账五个大饼”。
- 被直接艾特时贴着对方的问题短回，不走长篇标准问答。

程序不硬拼对外话术。群聊发言、黑话使用、内线私信、内线回复压缩都交给 LLM 处理；代码只负责上下文、节流、记忆和质量过滤。

主动插话触发不等于立即回复。插件会随机等一小段时间，到点后重新拿最新聊天判断是否还值得说，避免过了很久才回复旧话题。

当遇到账号插件行为相关问题时，插件会倾向于生成短句解释，例如：

```text
我就拿这号说两句
插件那套别问我
那触发跟我没啥关系
```
